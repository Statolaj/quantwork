# quantwork
A repository dedicated to simple programming tasks relevant for me in computational finance.

The current list of files includes:
  1) dice_game.py which implements an optimal strategy for stopping a dice game where you roll a six-sided die for a fixed number of rounds. The goal is maximizing the expected number of pips. Uses recursion. 
  2) gaussian_moments.py which simulates the standard Gaussian $m^{th}$ moment using Monte Carlo. The files contains 5 different version: A) Simple Monte Carlo, B) Importance Sampling from variance $m+1$, C) Importance Sampling from variance $m+1$ and antithetic variables, D) Importance sampling using $\Gamma(\tfrac{m+1}{2},2)$-distribution and antithetic variables, E) Mixture of two Gaussian at $\pm \sqrt{m}$ and antithetic variables. The best performing version is D, where essentially the variance is 0 of the estimator.
  3) heat_equation_solver.py which implements a finite difference method for solving the heat equation with a boundary condition in dimensions 1 and 2. The method can be extended to higher dimension, but since the number of grid points grows exponentially in the dimension, it is not possible to apply this method in practice. Instead use Feynmann-Kac to and simulate the expectation using Monte Carlo, where the precision does not depend on the dimension.  
  4) pi_estimation.cpp which implements a simple Monte Carlo estimation of pi in C++ using uniform samples from a 2x2 square and computing the fraction that land inside the unit circle. With uniform samples, the acceptance rate is pi/4 = 78.5%, but an idea to improve this is using importance sampling and switching to e.g. $2*\beta(2,2)$-distribution so generally points are closer to the center. 
  5) black_scholes_pricing_from_heat_equation.pdf contains a short derivation from the Black-Scholes partial differential equation to a heat equation, and how the solution of this equation can be used to price the original option. Emphasis is placed on the European call option $\Phi(S_T)=(S_T-K,0)^+$ and digital call option $Phi(S_T)=\mathbf{1}\{S_T>K\}$.
  6) black_scholes_pricing_from_heat_equation.py contains an extended heat equation solver compared to heat_equation_solver.py, which can handle other boundaries than 0. In particular we implement the ideas in black_scholes_pricing_from_heat_equation.pdf and numerically solve the pricing PDEs stemming from the european and digital call options. We place the grid such that $\log(K)$ is the center grid point in both cases. The file contains a test case of $S_0 = 200$, $K = 200$, $T = 1$, $ r = 5$ and $\sigma = 0.5$. The option price is plotted against the stock price on the grid and the deviation for both types of options at $T$ compared to the theoretical closed form pricing formulas are less than $0.002$ with the closed form price being $35.925$ and $0.349$ for european and digital, respectivily.
